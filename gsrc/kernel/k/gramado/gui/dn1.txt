
    Design Note 1.


    Design Note sobre o módulo /gui que pertence ao executive do kernel base.

    gui pode ser o nome da classe que contem todos os recursos
gráficos. ou pode ser awt.

    Diversas primitivas gráficas são oferecidas no kernel base em função do ganho de desempenho. 
	Pode haver funções mas elaboradas em user mode que se utilizem dessas primitivas. 
	Aliás, em user mode terá um gerente de recursos gráficos, que chamará as primitivas 
	contidas aqui nesse módulo em kernel mode.
    Os processos em user mode utilizarão os recursos oferecidos pelo gerente
de recursos gráficos presente em user mode. Já a chamada direta aos recursos gráficos
oferecidos pelo kernel base serão de uso privilegiado, somente alguns processo e
servidores.
    Cada janela tem que ter seu próprio buffer. Assim, o conteúdo de um buffer contendo
uma janela pintada, ou uma imagem pode ir direto para o LFB, sem passar pelo backbuffer.
Isso provavelmente se chama janelas flutuantes. Mas existem elementos gráficos
que são pintados diretamente no backbuffer, como a ima´gem de pano de fundo.


 Sobre buffers:
 ==============
   
   Front Buffer:
       O buffer que aparece na tela. Também chamado de on-screen buffer ou LFB.
   
   Back Buffer:
       O buffer que não aparece na tela. Também chamado de off-screen buffer.


SwapBuffers é a função que copia o conteúdo do back buffer no front buffer.
