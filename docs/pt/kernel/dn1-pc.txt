Sobre o microkernel e o process manager:
========================================

    /pc process control ( ipc, scheduler, mm )


    O Process Manager é uma parte fundamental do kernel base. Seu arquivo inicial é process.c, 
os outros arquivos são complemento do process manager.
    O process manager faz parte do microkernel.


    *IMPORTANTE: 
     As rotinas nesse módulo são em parte extremamente dependente do hardware. Deve-se criar alguma abstração e encapsulamento, assim como no módulo hal.    

    Obs: 
    schedi.c - Comtém funções que complementam sched.c.
    
    Obs: 
    threadi.c - Comtém funções que complementa thread.c. 


Sobre o gerenciamento de Heap:
=============================
    O arquivo pheap.c gerencia heaps para processos e o arquivo theap.c gerencia heaps para threads.
    

Sobre o gerenciamento de Stacks:
================================
    O arquivo tstack.c gerencia stasks para threads.
